Gridiron AI is an AI-powered fantasy football prediction platform that reverse-engineers Vegas sportsbook prop probability curves into player-level PPR fantasy point distributions. Every player, every week: a full probability distribution showing floor, median, ceiling, boom probability, and bust probability — derived from the sharpest prediction market in sports.

CORE THESIS
Vegas prices the full probability distribution of every NFL player's weekly outcomes through its alt line ladder markets. Gridiron AI strips the bookmaker's margin, reconstructs the distribution curve, runs a Monte Carlo simulation, and delivers the market's own probability — translated into PPR fantasy points. No existing consumer fantasy tool does this.

THE PRODUCT — HOW IT WORKS

The Core Methodology
Every major sportsbook prices a full set of player props for every NFL game, every week. The alt yard ladder — rungs at 25+, 40+, 50+, 60+, 75+, 80+, 90+, 100+, 125+, 150+, 175+, 200+ receiving yards — is not a collection of separate bets. It is a complete cumulative distribution function of that player's receiving yards outcomes for this game, priced by the sharpest analysts in sports with the most current information available anywhere.

Gridiron AI reads that distribution, strips the bookmaker's margin, fits a smooth curve through the data points, and converts it to PPR fantasy points via Monte Carlo simulation. The output is a full probability distribution: floor, P25, median, P75, ceiling, boom%, bust% — for every player, every week.

The Weekly Engine — 10 Phases
Phase
What Happens
Phase 1 — Game Lines
Pull game total, spread, team totals. Identify sharp books (Pinnacle at 2x weight). Compute opening vs. current line movement.
Phase 2 — Team Envelope
Convert implied team total to passing yards, pass attempts, rush attempts, red zone trips. Apply game script adjustment from spread (0.8% pass rate shift per point of spread).
Phase 3 — Prop Ingestion
Pull all player props: primary yards/receptions/TD lines, full alt ladders, combo props, longest reception, head-to-head, team share. Detect prop suspensions vs Tuesday baseline — suspension = PRIORITY ALERT immediately.
Phase 4 — Vig Removal
Positive odds: P_raw = 100/(odds+100). Negative odds: P_raw = |odds|/(|odds|+100). Two-sided vig removal for primary lines. One-sided formula (P_raw/1.10) for alt lines. Cross-book consensus = median across active books, Pinnacle at 2x weight.
Phase 5 — CDF Construction
Build cumulative distribution arrays from alt ladder consensus probabilities. Validate monotonic decrease. Fit smooth curve via PCHIP interpolation (scipy). Extract median, floor (10th pct), ceiling (90th pct), boom%, bust% directly.
Phase 6 — Adjustments
Apply longest reception ceiling adjustment: ADJ = (line - 22) x 0.018, cap at +40%/-20%. Compute combo prop correlation signal. Flag reverse line movement and juice shifts.
Phase 7 — Monte Carlo
Draw 100,000 correlated scenarios. CORR_COEFF = clamp(0.65 + corr_signal x 2.0, 0.40, 0.90). PPR_i = (yards/10) + (rec x 1.0) + (TDs x 6). Read floor, median, ceiling, boom%, bust% from sorted output.
Phase 8 — Internal Model
Parallel model using role data: TARGETS_PROJ = ADJ_PASS_ATT x TARGET_SHARE x (1 + FIRST_READ_SHARE x 0.12). Apply Season Budget adjustment, TD debt tracker (TD_DEBT x 0.20, cap +/-0.40), DNA volatility multiplier by grade (S+:0.72 through X:1.65).
Phase 9 — Final Blend
FINAL_X = (MARKET_X x MKT_CONFIDENCE) + (INTERNAL_X x (1-MKT_CONFIDENCE)) for each percentile. Apply line move shift (yards/10 x timing_weight, cap +/-3.0 PPR). Apply Bayesian update: BAYESIAN_MEDIAN = (FINAL_MEDIAN x PRIOR_WT) + (L4_FPPG x DATA_WT).
Phase 10 — Post-Game
Monday 8AM: pull actuals, compute PREDICTION_ERROR = ACTUAL - FINAL_MEDIAN. Tag failure mode (TD_VARIANCE, VOLUME_MISS, ROLE_MISS, GAME_SCRIPT_MISS, etc.). Update season budget tracker and Bayesian weights.

Proprietary Layer — What We Add on Top of Vegas
Layer
What It Does
DNA Volatility Calibration
Players graded S+ through X. SD multipliers: S+=0.72, S=0.86, B=1.00, V=1.18, H=1.38, X=1.65. A boom/bust player gets a wider distribution than Vegas implies.
Season Budget System
Preseason projected point pool tracked weekly. BUDGET_ADJ = BUDGET_PACE_DEV x DNA_COEFF x (1/SQRT(WEEKS_REM)) x -1.0. Cap: -20% max reduction, +15% max boost.
Role Change Detection
Snap share and target share shifts detected 48–72 hours before prop market adjusts. If SNAP_SHARE or TARGET_SHARE changed >= 15%, PRIOR_WEIGHT resets to 0.70 regardless of week number.
Earned vs Gifted Classification
Recent scores tagged as earned (volume-driven) or gifted (TD-dependent). TD-dependent boom risk flagged on player card.
Confidence Calibration Display
Model tracks its own historical accuracy by confidence tier. If historical accuracy diverges from formula by >5%, DISPLAYED_CONFIDENCE = historical_accuracy_at_tier. Shows what the model has actually achieved.

Core Components to Build
Component
Spec
Prop ingestion pipeline
Scheduled pull from The Odds API on fixed weekly cadence (Tues 6AM through Sun 7:30AM ET). Parse all player props by type, store in structured DB. Compare Sunday AM vs Tuesday baseline to detect suspensions.
Vig removal engine
Positive/negative odds conversion. Two-sided formula for primary o/u lines. One-sided formula (P_raw/(1+vig_rate)) for alt lines with 8% for primary books, 10–12% for secondary. Pinnacle at 2x weight in median.
CDF fitting module
PCHIP interpolation through validated alt ladder points (scipy.interpolate.PchipInterpolator in Python). Enforce monotonic decrease. Boundary conditions: CDF(0)=1.0, CDF(inf)=0.0. Linear interpolation fallback if fewer than 6 rungs.
Monte Carlo simulator
100,000 correlated scenario draws. Cholesky decomposition or Gaussian copula for yards/receptions correlation. PPR = (yards/10) + rec + (TDs x 6). Sort output, read all percentiles directly.
Season Budget tracker
Per-player point pool initialized preseason. Weekly actual update. Pace deviation calculation. Regression/bounce-back signal output. Reset protocol for role changes.
Internal fallback model
Role distribution model for players with thin/missing prop markets. Gamma distribution parameterized by DNA grade. Used when MKT_CONFIDENCE < 0.50 or prop data unavailable.
Blending engine
Percentile-by-percentile blend: FINAL_X = (MARKET_X x MKT_CONF) + (INTERNAL_X x (1-MKT_CONF)). Additive line move shift. Additive juice shift signal. Bayesian median update.
Player card UI
Floor / P25 / Median / P75 / Ceiling bars. Boom% and Bust% prominently displayed. Probability histogram of Monte Carlo output. Market Confidence indicator. Vegas intel summary in plain English. DNA grade badge.
Start/Sit comparison view
Two-player side-by-side. Median differential, EDGE classification (Strong >= 3.0 PPR / Moderate 1.0–3.0 / Lean Only < 1.0). Distribution overlap visualization. Prop suspension alert if applicable.
Post-game processing loop
Monday 8AM automated run: pull actuals from stats feed, compute errors, tag failure modes from taxonomy (TD_VARIANCE / VOLUME_MISS / ROLE_MISS / INJURY_MISS / GAME_SCRIPT_MISS / WEATHER_MISS / MARKET_MISS / CORRECT_PROCESS), update all trackers.

BUILD PHILOSOPHY
The AI reads a market that has already predicted outcomes. It does not generate predictions from scratch. The computational load is modest: weekly batch pulls, vig removal arithmetic, curve fitting, and scheduled Monte Carlo runs. This is not a real-time system. No live data required during games.
